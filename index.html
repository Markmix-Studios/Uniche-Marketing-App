UNICHE Backend API Documentation

Complete API reference for implementing UNICHE backend services

🔐 Authentication

POST /api/auth/login Login user 
{
  "email": "student@ug.edu.gh",
  "password": "password123",
  "university": "ug"
}

Response: JWT token, user profile, university info

POST /api/auth/register Register new user 
{
  "email": "newstudent@knust.edu.gh",
  "password": "securepass",
  "fullName": "John Doe",
  "university": "knust",
  "year": "2",
  "phoneNumber": "+233501234567"
}
POST /api/auth/google Google OAuth login 
{
  "googleToken": "ya29.a0AfH6SMC...",
  "university": "ashesi"
}

🛍️ Marketplace

GET /api/items Get marketplace items

Query Parameters:

?university=ug&category=electronics&search=iphone&page=1&limit=20&sort=recent
POST /api/items Create new listing 
{
  "title": "iPhone 13 Pro Max",
  "description": "Excellent condition, barely used",
  "price": 3500.00,
  "category": "electronics",
  "images": ["image1.jpg", "image2.jpg"],
  "isUrgent": false,
  "location": "UG Campus"
}

👤 User & Messaging

GET /api/users/profile Get user profile

Returns: user info, ratings, active listings

POST /api/messages Send message 
{
  "recipientId": "user123",
  "itemId": "item456",
  "message": "Is this still available?"
}

🚀 Quick Implementation Guide

Recommended Tech Stack

  • • Backend: Node.js + Express
  • • Database: MongoDB with Mongoose
  • • Auth: Firebase Auth or JWT
  • • Storage: Firebase Storage
  • • Real-time: Socket.io for messaging

Security Considerations

  • • University email verification
  • • Rate limiting on API endpoints
  • • Image upload validation
  • • User reporting system
  • • Content moderation
